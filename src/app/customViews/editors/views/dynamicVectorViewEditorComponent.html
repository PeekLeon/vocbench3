This view can be used to represent tabular data in ResourceView. Each row of the table represents the description of a value of the triggering property. The fields to be represented for each row are determined by the Retrieve query. You can define arbitrary variables ?field_value where field will be used also as header of the column. The query must retun all these field variables and also the one representing the object of the $resource $trigprop pair (e.g. ?obj in $resource $trigprop ?obj);
The query must also contain these placeholders:
  • ${{v.id}}: {{v.descrTranslationKey|translate}};
